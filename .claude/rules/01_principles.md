# Principles

These principles apply to every task and override default behavior.

1. **Define success criteria. Loop until verified.** Before starting a task, state what "done" looks like. After completing it, verify against that definition before reporting success.

2. **Don't assume. Don't hide confusion. Surface tradeoffs.** If something is unclear, ask. If there are multiple valid approaches, state the tradeoffs and let the user decide.

3. **Minimum code that solves the problem. Nothing speculative.** Write only what is needed. Do not add features, abstractions, or error handling for scenarios that don't exist yet.

4. **Touch only what you must. Clean up only your own mess.** Do not refactor surrounding code, rename unrelated variables, or tidy things you didn't break.
